# cc-session-dashboard

`~/.claude/projects/` 配下の Claude Code セッションログ (JSONL) を解析し、
プロジェクト毎・セッション毎のトークン消費・キャッシュ利用・概算コストを
ローカルで可視化するダッシュボード。

## 主な機能

- **ダッシュボード**: 全プロジェクトの概算コスト・トークン集計、日次コストのモデル別チャート
- **プロジェクト画面**: セッション一覧（タイトル・モデル・最大コンテキスト・コスト）
- **セッション詳細**: ターンが進む度のコンテキストウインドウ推移を二段重ねの棒グラフで表示
  - 上段: コンテキスト内訳（キャッシュ読込 / キャッシュ書込 / 非キャッシュ入力）の積み上げバー
  - 下段: 出力トークン（コンテキストの構成要素ではないため独立表示。X 軸・ズームは上段と連動）
  - コンパクション境界（pre → post トークン）の表示
  - 対話トランスクリプトとチャートの相互同期（チャートをクリックすると該当メッセージへスクロール）
  - トランスクリプト右に 90 度回転した縦型チャート（Y 軸 = リクエスト順）を sticky 配置し、
    スクロールに連動して表示中のターンを帯でハイライト。バーにホバーすると
    そのターンを開始したプロンプトのプレビューを表示
  - サブエージェント（sidechain / `subagents/*.jsonl`）の表示切替
- **ライブ更新**: 進行中セッションのログ追記を `fs.watch` + SSE で検知し自動反映

## 起動方法

Node.js 22+ が必要。

```sh
npm install
npm run dev    # http://localhost:5173 (API: 3000)
```

本番モード（1 プロセス・ビルド済み静的ファイル配信）:

```sh
npm run build
npm start      # http://localhost:3000
```

## テスト

```sh
npm test        # パーサ・集計・コスト計算のユニットテスト
npm run typecheck
```

## コストについて

表示されるコストは **API 従量課金に換算した概算**です。Pro / Max などの
サブスクリプションで利用している場合の実費ではありません。

- モデル別単価は [`pricing.json`](./pricing.json) で管理（前方一致で解決）
- キャッシュ書込は 5m TTL = input 単価 × 1.25、1h TTL = × 2.0、キャッシュ読込 = × 0.1
- 料金表に無いモデルはコスト 0 として扱い、画面上に警告を表示する

## 実装メモ

- 同一 API リクエストはストリーミング中に同じ `requestId` で複数行記録されるため、
  `requestId`（無ければ `message.id` → `uuid`）で重複排除して集計する
- リクエスト時点の実効コンテキストサイズ = `input + cache_read + cache_creation`
- 解析結果は mtime + size で無効化されるインメモリキャッシュに保持し、
  追記分のみ増分パースするため再スキャンは軽量
- ログディレクトリは環境変数 `CLAUDE_PROJECTS_DIR` で変更可能（既定: `~/.claude/projects`）
